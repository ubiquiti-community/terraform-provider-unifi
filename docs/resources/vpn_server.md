# Vpn Server (Resource)

`unifi_vpn_server` manages VPN server configurations (WireGuard, L2TP, or OpenVPN) in the UniFi controller.

## Example Usage

```terraform
# A VPN server is configured with exactly one of the `wireguard`, `l2tp`, or
# `openvpn` nested blocks. The block you choose determines the server type.
# The `subnet` is the server's tunnel network in gateway form (the first
# address, e.g. `10.100.0.1/24`, is the server's tunnel IP).

# WireGuard server. If `private_key` is omitted the provider generates one at
# create time and exposes the derived `public_key` as a computed attribute
# (useful for configuring `unifi_wireguard_peer` resources).
resource "unifi_vpn_server" "wireguard" {
  name   = "wireguard"
  subnet = "10.100.0.1/24"

  wireguard = {
    port = 51820
  }
}

# WireGuard server that also pushes custom DNS servers to connecting clients and
# binds to a specific WAN interface. `dns.enabled` defaults to true when
# `servers` is non-empty.
resource "unifi_vpn_server" "wireguard_dns" {
  name   = "wireguard-dns"
  subnet = "10.101.0.1/24"

  wireguard = {
    port = 51821
  }

  dns = {
    servers = ["1.1.1.1", "1.0.0.1"]
  }

  wan = {
    ip        = "any"
    interface = "wan"
  }
}

# L2TP/IPsec server. `pre_shared_key` is required by the controller. L2TP
# servers authenticate users against a RADIUS profile.
resource "unifi_radius_profile" "l2tp" {
  name = "l2tp-radius"

  auth_server {
    ip     = "192.168.1.100"
    port   = 1812
    secret = "radius-secret"
  }
}

resource "unifi_vpn_server" "l2tp" {
  name             = "l2tp"
  subnet           = "10.110.0.1/24"
  radiusprofile_id = unifi_radius_profile.l2tp.id

  l2tp = {
    pre_shared_key     = "change-me-to-a-strong-secret"
    allow_weak_ciphers = false
  }
}

# OpenVPN server. The controller generates the certificates and keys, which are
# exposed as computed (sensitive) attributes. `mode` is one of `server` or
# `site-to-site`; `encryption_cipher` is one of `AES_256_GCM`, `AES_256_CBC`,
# or `BF_CBC`.
resource "unifi_vpn_server" "openvpn" {
  name             = "openvpn"
  subnet           = "10.120.0.1/24"
  radiusprofile_id = unifi_radius_profile.l2tp.id

  openvpn = {
    port              = 1194
    mode              = "server"
    encryption_cipher = "AES_256_GCM"
  }
}
```

## Schema

### Required

- `name` (String) The name of the VPN server.
- `subnet` (String) The VPN server subnet in CIDR notation (e.g., `10.100.0.1/24`). The first address is the server's tunnel IP, so use the gateway form (`10.100.0.1/24`) — the network address (`10.100.0.0/24`) is rejected with `api.err.IncorrectIPSubnetSpec`.

### Optional

- `dns` (Attributes) DNS configuration pushed to VPN clients. (see [below for nested schema](#nestedatt--dns))
- `enabled` (Boolean) Specifies whether the VPN server is enabled.
- `l2tp` (Attributes) L2TP VPN server configuration. Exactly one of `wireguard`, `l2tp`, or `openvpn` must be specified. (see [below for nested schema](#nestedatt--l2tp))
- `openvpn` (Attributes) OpenVPN server configuration. Exactly one of `wireguard`, `l2tp`, or `openvpn` must be specified. (see [below for nested schema](#nestedatt--openvpn))
- `radiusprofile_id` (String) The ID of the RADIUS profile to use for authentication. Applicable to L2TP and OpenVPN server types.
- `site` (String) The name of the site to associate the VPN server with.
- `wan` (Attributes) WAN binding configuration for the VPN server. (see [below for nested schema](#nestedatt--wan))
- `wireguard` (Attributes) WireGuard VPN server configuration. Exactly one of `wireguard`, `l2tp`, or `openvpn` must be specified. (see [below for nested schema](#nestedatt--wireguard))

### Read-Only

- `id` (String) The ID of the VPN server.

<a id="nestedatt--dns"></a>
### Nested Schema for `dns`

Optional:

- `enabled` (Boolean) Specifies whether custom DNS servers are enabled for VPN clients. Defaults to `true` when `servers` is non-empty.
- `servers` (List of String) DNS servers to push to VPN clients.


<a id="nestedatt--l2tp"></a>
### Nested Schema for `l2tp`

Required:

- `pre_shared_key` (String, Sensitive) IPsec pre-shared key for L2TP. Required by the UniFi controller.

Optional:

- `allow_weak_ciphers` (Boolean) Allow weak ciphers for L2TP connections.


<a id="nestedatt--openvpn"></a>
### Nested Schema for `openvpn`

Optional:

- `encryption_cipher` (String) Encryption cipher for OpenVPN.
- `mode` (String) OpenVPN mode.
- `port` (Number) Port for the OpenVPN server to listen on.

Read-Only:

- `auth_key` (String, Sensitive) OpenVPN static authentication key generated by the controller.
- `ca_crt` (String, Sensitive) CA certificate generated by the controller.
- `ca_key` (String, Sensitive) CA private key generated by the controller.
- `dh_key` (String, Sensitive) Diffie-Hellman parameters generated by the controller.
- `server_crt` (String, Sensitive) Server certificate generated by the controller.
- `server_key` (String, Sensitive) Server private key generated by the controller.
- `shared_client_crt` (String, Sensitive) Shared client certificate generated by the controller.
- `shared_client_key` (String, Sensitive) Shared client private key generated by the controller.


<a id="nestedatt--wan"></a>
### Nested Schema for `wan`

Optional:

- `interface` (String) WAN interface to use for the VPN server (e.g., `wan`, `wan2`).
- `ip` (String) Local WAN IP to bind the VPN server to. Use `any` to listen on all addresses.


<a id="nestedatt--wireguard"></a>
### Nested Schema for `wireguard`

Optional:

- `port` (Number) UDP port for the WireGuard server to listen on.
- `private_key` (String, Sensitive) WireGuard private key for this server (base64). If not specified, the provider generates one at create time (the controller does not generate it and rejects a server without a key).

Read-Only:

- `public_key` (String) WireGuard public key for this server. Computed from the private key.

## Import

Import is supported using the following syntax:

The [` + "`" + `terraform import` + "`" + ` command](https://developer.hashicorp.com/terraform/cli/commands/import) can be used, for example:

```shell
# VPN servers can be imported by their 24-character hex network ID:
terraform import unifi_vpn_server.wireguard 5f3e1c2b9a8d7e6f5a4b3c2d

# Or by name, using the "name=" prefix:
terraform import unifi_vpn_server.wireguard name=wireguard

# Both forms may be prefixed with a site name and a colon to target a specific
# site (otherwise the provider's default site is used):
terraform import unifi_vpn_server.wireguard default:5f3e1c2b9a8d7e6f5a4b3c2d
terraform import unifi_vpn_server.wireguard default:name=wireguard
```
